🛠️ บันทึกทางเทคนิค: Identity Matrix (np.eye)

Identity Matrix ( \(I\) ) คือ Matrix ที่มีค่าเป็น 1 ในแนวทะแยงและ 0 ในตำแหน่งอื่นทั้งหมด มีคุณสมบัติพิเศษคือเมื่อนำไปคูณกับข้อมูลใดๆ ข้อมูลนั้นจะไม่เปลี่ยนแปลง (Identity Mapping)

\[ I = \begin{bmatrix} 1 & 0 & 0 \\ 0 & 1 & 0 \\ 0 & 0 & 1 \end{bmatrix} \]
การใช้งานในโปรเจกต์:

เราใช้ np.eye(d_model) เพื่อเริ่มต้นค่า Weight ของ Transformer แทนการสุ่ม ซึ่งช่วยให้ AI "มองเห็น" ข้อมูลดั้งเดิมได้ชัดเจนที่สุดโดยไม่ถูกบิดเบือน

💻 ตัวอย่างการทำงานใน Python
import numpy as np
I = np.eye(3)
vector = np.array([5, 2, 9])

# ผลลัพธ์: [5, 2, 9] (ข้อมูลคงเดิม)
result = vector @ I

การเปลี่ยนข้อความเป็นตัวเลข (Statistical Representation)

เราสร้างความหมายเบื้องต้นจากการสังเกตพฤติกรรมการใช้คำในประโยค โดยนับการปรากฏตัวร่วมกันเพื่อสร้าง "พิมพ์เขียว" ทางสถิติ

\[C_{i,j} = \sum_{s \in S} \text{interaction}(w_i, w_j)\]
รายละเอียดตัวแปร:
  • C : ตารางสรุปความถี่ทั้งหมด (มาจากผลรวมสถิติ)
  • i, j : ตำแหน่งดัชนีของคำในคลังศัพท์ (ดึงมาจาก Vocabulary)
  • S : ชุดประโยคจากไฟล์ corpus.txt (แหล่งข้อมูลดิบ)
Input Data
"cat sits on mat"
"car drives on road"
Frequency Matrix
cat -> [sits: 1, mat: 1]
car -> [drives: 1, road: 1]

การลดมิติข้อมูล (Dimensionality Reduction)

เราใช้การแยกส่วน Matrix เพื่อบีบอัดข้อมูลที่กระจัดกระจายให้เหลือเพียงเวกเตอร์ 8 มิติที่เข้มข้นไปด้วยความหมาย

\[A = U \Sigma V^T\]
รายละเอียดตัวแปร:
  • A : ข้อมูล Matrix ความถี่ (มาจากขั้นตอนการสร้างพิมพ์เขียว)
  • U : แก่นของเวกเตอร์คำศัพท์ที่เรานำไปใช้ (Word Embeddings)
  • Σ : ระดับความสำคัญของแต่ละคุณลักษณะที่ถูกค้นพบ

🎯 Analysis: การวิเคราะห์ความสัมพันธ์

การคำนวณระยะห่าง (Semantic Proximity)

เพื่อระบุว่าคำไหนคือ "ครอบครัวเดียวกัน" เราวัดระยะห่างระหว่างจุดในพื้นที่ 8 มิติ ยิ่งระยะห่างน้อย ความหมายยิ่งใกล้กัน

\[d = \sqrt{\sum (q_i - p_i)^2\]
รายละเอียดตัวแปร:
  • d : ระยะห่างทางคณิตศาสตร์ (ใช้เพื่อตัดสินใจจัดกลุ่ม)
  • p, q : พิกัดเวกเตอร์ของคำที่นำมาเปรียบเทียบ (จาก learned_vectors.json)
Comparison Logic
Intra-Group: "cat" vs "dog"
Inter-Group: "cat" vs "car"
Calculated Distance
Close (0.00) -> [Group A]
Far (1.15) -> [Separate]

🧠 Intelligence: ความเข้าใจตามบริบท

กลไกความใส่ใจ (Self-Attention)

นี่คือจุดที่ทำให้เวกเตอร์ "มีชีวิต" โดยการปรับเปลี่ยนความหมายตามคำรอบข้าง ทำให้คอมพิวเตอร์เข้าใจบริบทของประโยคได้จริง

\[\text{Attn}(Q, K, V) = \left( \frac{\exp(QK^T / \sqrt{d_k})}{\sum \exp(QK^T / \sqrt{d_k})} \right) V\]
รายละเอียดตัวแปร:
  • Q (Query) : คำถามจากเวกเตอร์ดั้งเดิม ($\times W_Q$) - คำนี้ต้องการมองหาอะไร?
  • K (Key) : ข้อมูลระบุตัวตน ($\times W_K$) - คำอื่นมีข้อมูลอะไรให้บ้าง?
  • V (Value) : เนื้อหาความหมายจริง ($\times W_V$) - ข้อมูลที่จะถูกส่งต่อให้ Q
  • \(\exp\) : ฟังก์ชัน Exponential (เลขชี้กำลังของ e) เพื่อขยายความต่างของคะแนน
📖 ส่วนอ้างอิง (Reference):

ส่วนของสมการ \(\frac{\exp(\dots)}{\sum \exp(\dots)}\) คือการคำนวณ Softmax ในรูปของสูตรคณิตศาสตร์ดิบ ซึ่งทำหน้าที่แปลง "คะแนนดิบ" (Raw Scores) ให้กลายเป็น "น้ำหนักความสนใจ" (Attention Weights) ที่มีค่าระหว่าง 0 ถึง 1 และผลรวมของทั้งประโยคต้องเท่ากับ 1 พอดี

⚠️ บันทึกสำคัญ: โครงสร้าง (Architecture) vs การฝึกฝน (Training)

การมีสถาปัตยกรรม Transformer ไม่ได้ทำให้ AI "ฉลาด" โดยอัตโนมัติ AI จะฉลาดได้ต้องผ่านขั้นตอน Training เพื่อปรับค่า Matrix น้ำหนัก ($W_{pred}$) ให้ตรงกับข้อมูลใน corpus.txt หาก AI ทำนายผลออกมาซ้ำๆ หรือดูไม่สมเหตุสมผล นั่นแปลว่ามันยังเรียนรู้จากข้อมูลไม่พอ (Underfitting)
